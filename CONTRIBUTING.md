# Contributing to Crux

This repository is the Crux framework source.
It is not the same thing as an installed Crux runtime inside a user project.

If you are editing agents, skills, templates, workflows, or scripts here, keep the
source layout and installed layout separate in your head.

## Two Layouts

### Source Layout

This repository uses the framework source layout:

```text
/
├── AGENTS.md
├── COORDINATOR.md
├── agents/
├── skills/
├── templates/
├── workflows/
├── bus/
├── scripts/
├── docs/
└── README.md
```

You edit framework content here.

### Installed Layout

A user project runs Crux in this shape:

```text
.crux/
├── AGENTS.md
├── COORDINATOR.md
├── agents/
├── skills/
├── templates/
├── workflows/
├── bus/
├── docs/
├── summaries/
└── workspace/
```

This is the only valid runtime and documentation model.

## Mandatory Rules

1. Source repo layout is for framework development only.
2. Installed layout is the only valid runtime/documentation model.
3. All architecture documents must reference installed paths.
4. Install script is responsible for mapping source layout into installed `.crux/`.
5. Generated docs, summaries, workspace, and user outputs must never be committed as source framework content.

## Path Rule

Even though you edit files here as:

```text
agents/backend-developer/AGENT.md
skills/api-surface-analyser/SKILL.md
workflows/tenant-onboarding.md
```

the text inside docs and architecture files should still refer to:

```text
.crux/agents/backend-developer/AGENT.md
.crux/skills/api-surface-analyser/SKILL.md
.crux/workflows/tenant-onboarding.md
```

Reason:
- contributors edit the source layout
- users and agents operate against the installed layout

Do not document source paths as if they were runtime paths.

## What Install Does

`scripts/install.sh` copies framework source into a user project's `.crux/` directory.

Copied by install:
- `AGENTS.md` → `.crux/AGENTS.md`
- `COORDINATOR.md` → `.crux/COORDINATOR.md`
- `agents/*` → `.crux/agents/*`
- `skills/*` → `.crux/skills/*`
- `templates/*` → `.crux/templates/*`
- `workflows/*` → `.crux/workflows/*`
- `bus/*` → `.crux/bus/*`

Not generated by install:
- `.crux/CONSTITUTION.md`
- `.crux/SOUL.md`
- `.crux/workspace/*`
- `.crux/docs/*`
- `.crux/summaries/*`
- project decisions
- user output trees such as `notes/`, `operations/`, or `docs/compliance/`

Those are created during onboarding, first boot, or lazy-loading.

## What Convert Does

`scripts/convert.sh` reads either:
- the source repo layout in this repository, or
- an installed `.crux/` layout in a user project

and then writes tool-specific files for:
- OpenCode
- Claude Code
- Cursor

Typical framework-repo usage:

```sh
./scripts/convert.sh
```

Typical user-project usage:

```sh
./scripts/convert.sh --source .crux
```

## Editing Guidelines

### Agents

When adding or updating an agent:
- edit `agents/{role}/AGENT.md`
- edit `agents/{role}/onboarding.md`
- keep agent-specific generated source material under `agents/{role}/assets/`
- if the agent needs reusable generated docs, those docs should be created from agent assets, not committed as ready-made runtime docs

### Skills

When adding or updating a skill:
- create or edit `skills/{name}/SKILL.md`
- the folder name and `name:` frontmatter must match
- reference installed runtime paths inside the file

### Workflows

Use committed `workflows/` only for reusable framework workflows.

If a workflow belongs to one agent package and should be generated on demand:
- keep the source under `agents/{role}/assets/`
- generate the runtime file into `.crux/workflows/{name}.md` during onboarding or first use

Example:
- source: `agents/red-team-lead/assets/pentest-engagement.workflow.template.md`
- runtime: `.crux/workflows/pentest-engagement.md`

### Generated Files

Do not commit:
- `.crux/workspace/*`
- `.crux/docs/*`
- `.crux/summaries/*`
- project-specific decisions
- user-space outputs produced during onboarding or operations

If you find yourself about to commit generated runtime content, stop and move the source into:
- `templates/`, or
- `agents/{role}/assets/`

## Quick Checks Before You Finish

Before closing a change:

1. Make sure runtime docs still reference installed `.crux/...` paths.
2. Make sure source-only explanations live in `CONTRIBUTING.md` or `docs/source-layout.md`, not in runtime-facing docs.
3. Run:

```sh
./scripts/convert.sh --dry-run --tool opencode
./scripts/convert.sh --dry-run --tool claude-code
```

4. If you changed installer behaviour, also sanity check:

```sh
bash -n scripts/install.sh
bash -n scripts/convert.sh
bash -n scripts/update.sh
```
