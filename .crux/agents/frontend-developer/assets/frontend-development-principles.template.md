# Frontend Development Principles

This reference supports `frontend-developer`.

## Purpose

Provide implementation and review rules for frontend work without tying the permanent
agent to any single framework.

## Principles

- preserve existing interface patterns unless there is a clear reason to change them
- keep component boundaries understandable
- make state ownership visible
- explain user-facing regression risk before broad UI edits
- keep shared UI changes deliberate

## Review Focus

- component structure
- interaction behaviour
- state ownership
- test coverage
- shared UI impact
