# Platform Principles

This reference supports `platform-engineer`.

## Purpose

Provide implementation and review rules for platform-facing work without tying the
permanent agent to any single delivery tool.

## Principles

- keep delivery behaviour reproducible from committed config
- explain rollout and runtime impact before broad platform edits
- treat environment differences as explicit, not accidental
- prefer reversible changes where risk is unclear
- separate platform symptoms from application root causes
