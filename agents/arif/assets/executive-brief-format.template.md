# Executive Brief Format

This template is the source material for `.crux/docs/executive-brief-format.md`.
Use it when `arif` needs to compress analysis into a concise recommendation
for leadership or client-facing decision discussions.

---

## Core Position

Executives rarely need more information first.
They need the right framing, the preferred path, and the main trade-off.

Default brief style:
- short
- direct
- low jargon
- decision-ready

---

## Standard Structure

```markdown
# Executive Recommendation

## Situation
- What is happening and why this matters now

## Recommendation
- The preferred path in plain language

## Why This Path
- 2-3 practical reasons only

## Main Trade-Off
- What we gain and what we accept

## Main Risk
- The risk that could change the decision

## Next Step
- The smallest action that moves this forward
```

---

## Tone Rules

- sound experienced, not theatrical
- avoid jargon unless it changes the decision
- do not try to impress with volume
- if assumptions are weak, say so directly

Preferred language:
- "The practical path is..."
- "The main trade-off is..."
- "I would start with..."
- "This only makes sense if..."

Avoid:
- inflated transformation language
- false certainty
- long option catalogs unless the user asked for them

---

## Length Guidance

Target:
- 5 to 12 lines for normal recommendations
- one short table only if comparison is essential

If the note becomes too long:
- cut background first
- keep the recommendation, trade-off, and next step

---

## Escalation Note

If the issue is not decision-ready, the brief should say so plainly:
- what is missing
- what cannot be assumed
- what one question or input would unlock the decision
