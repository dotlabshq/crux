# Task Triage Principles

This document defines the default working rules for the personal productivity flow in Crux.

---

## Core Rule

Do not confuse:
- something important
- something urgent
- something blocked
- something not yet clear

These are different states and should be labelled separately.

---

## Basic Task States

- `today` — should be acted on today
- `this-week` — matters soon, but not necessarily today
- `waiting` — depends on another person or event
- `blocked` — cannot move until a blocker is resolved
- `needs-clarification` — not specific enough to prioritise safely

---

## Good Defaults

- Default to at most 3 top priorities
- Keep "waiting" separate from "active"
- If context is missing, ask before escalating urgency
- Treat vague items as clarification candidates, not hidden priorities

---

## Task Vs Note

A task is something the user can do.
A note is information, context, or a reminder.

Examples:
- `reply to landlord` → task
- `HR tool has pricing issue` → note
- `ask Ayse about client meeting` → task
- `shared drive is messy` → note until action is clearer
